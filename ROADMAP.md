# Roadmap

## Current

- Public template extraction from a working internal prototype.
- Safe dry-run behavior by default.
- Telegram webhook validation support.
- Vertex AI Gemini integration.
- Local test scripts.

## Near Term

- Add GitHub Actions CI.
- Expand webhook edge-case tests.
- Add deployment screenshots or a step-by-step Cloud Run guide.
- Add structured configuration validation at startup.
- Add example prompts for common studio intake flows.

## Later

- Optional adapter interface for additional model providers.
- Optional queue-based processing for high-volume bots.
- Optional admin dashboard for routing rules.
- Terraform or OpenTofu example for infrastructure setup.
- More production observability examples.

## Non-Goals

- No hosted service.
- No paid platform dependency.
- No default production project IDs or tokens.
- No automatic Telegram webhook changes without operator confirmation.
